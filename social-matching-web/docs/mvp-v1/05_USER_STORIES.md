# 05 — User Stories

## 1. Participant Stories

### Discovery
- As a participant, I want to browse available gatherings so I can find one that feels right for me.
- As a participant, I want each event to feel clear and human so I can decide whether I want to apply.
- As a participant, I want to understand the vibe, city, timing, and process before applying.

### Trust and onboarding
- As a participant, I want the questionnaire to feel guided and respectful so that I do not feel evaluated.
- As a participant, I want to know why personal fields are being asked so that I can trust the process.
- As a participant, I want my social link to be used only for verification so I can feel safe sharing it.

### Apply
- As a participant, I want to apply to a specific event so that I can express interest in a gathering that feels right.
- As a participant, I want to avoid filling everything again for every event so that applying stays lightweight.
- As a participant, I want to know that payment only comes after approval so I do not feel pressured.
- As a participant, I want to see if I already applied so I do not submit duplicates.

### Status and follow-up
- As a participant, I want to know whether I am pending, approved, rejected, or waitlisted so I can understand what is happening.
- As a participant, I want to pay only after approval so that the process feels fair and clear.
- As a participant, I want to receive the relevant group/event information only when appropriate so the flow feels trustworthy.

### Event and feedback
- As a participant, I want to attend a small, curated gathering so that meeting new people feels less random.
- As a participant, I want to leave feedback after the event so the system can improve future groups.

---

## 2. Host Stories

- As a host, I want to submit or own an event so I can lead a gathering without building the whole system myself.
- As a host, I want the platform to help gather the right people so I do not have to manually recruit and screen alone.
- As a host, I want to receive an approved group rather than a raw applicant list so I can focus on the actual experience.
- As a host, I want the system to preserve the tone and quality of the gathering so the event feels intentional.
- As a host, I want post-event feedback so I can improve future gatherings.

---

## 3. Co-Host Stories

- As a co-host, I want to be attached to a specific event so I can support the host meaningfully.
- As a co-host, I want to receive only the context I need so that I can help without being overloaded.
- As a co-host, I want to support delivery of the event so the group experience runs smoothly.

---

## 4. Admin Stories

### Event governance
- As an admin, I want to approve or reject event proposals so I can maintain quality.
- As an admin, I want to control which events become public so the browse surface stays curated.

### Applicant review
- As an admin, I want to review event applicants so I can build good groups.
- As an admin, I want profile context, application context, and trust context in one place so I can make a confident decision.
- As an admin, I want to use algorithm recommendations without losing final control so I can combine signal with judgment.

### Group assembly
- As an admin, I want to approve, reject, or waitlist people so I can assemble balanced groups.
- As an admin, I want to handle replacement after missed payment so confirmed spots do not get wasted.

### Lifecycle and trust
- As an admin, I want to track attendance and feedback so I can improve the system over time.
- As an admin, I want internal trust-relevant information so I can protect the quality and safety of future events.

---

## 5. Platform Stories

- As the platform, I need to distinguish between profile readiness and event-specific apply state so that users are not forced to repeat onboarding unnecessarily.
- As the platform, I need to support a small number of parallel events so the MVP behaves like a platform, not a single-event funnel.
- As the platform, I need to keep the user journey clear and emotionally safe so people are willing to participate.
- As the platform, I need to preserve event quality over throughput so the pilot produces strong real-world outcomes.
